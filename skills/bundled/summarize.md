Summarize content from various sources.

## Usage

- `/summarize <url>` — Fetch the URL with `web_fetch` and provide a concise summary
- `/summarize <file-path>` — Read the file and summarize its contents
- `/summarize` (with pasted content) — Summarize the provided text

## Output Format

1. **Title/Source** — What was summarized
2. **Key Points** — 3-5 bullet points covering the main ideas
3. **Summary** — A 2-3 sentence synopsis
4. **Notable Details** — Any important specifics (dates, numbers, names)

## Guidelines

- Keep summaries concise and factual
- For code files, focus on purpose, architecture, and key functions
- For articles, focus on the thesis, supporting arguments, and conclusions
- For long documents, break into sections if needed
